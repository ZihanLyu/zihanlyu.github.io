Creative Part

  • I updated the getNeighborCoords() function.
    • Instead of storing values in a flat array, use an array of coordinate pairs, so it becomes:
      • Easier to read: Each neighbor is stored as [row, column], making it clearer.
      • Easier to iterate over: Instead of using index math (neighborCoords[i], neighborCoords[i+1]), now can just loop over pairs.
      • Less error-prone: No need to worry about indexing mistakes.
  • Implemented an update table function where users can adjust the table dimensions.
  • Added a dropdown tab with several popular patterns. Users can choose their desired template and add it to the center of the table.
